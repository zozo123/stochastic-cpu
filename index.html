The Inference Substrate.

A field report on the cloud that's being built underneath Software 3.0. From the agent-hour to the prompt-keyed cache to the three signatures on every 2030 merge.

by Yossi Eliaz · at incredibuild and islo.dev · Tel Aviv · 2026

01 Prelude ¶

The way we make software changed three times in a decade. We've only named two of them.

Software 1.0 was the code a human typed for a machine. Software 2.0 was the weights a dataset taught a machine to hold. Software 3.0 is the prompt — a program written in English that runs on a model that runs on a fleet. Karpathy gave us the first two names in 2017 and the third one in 2025, and the gap between those announcements is not an accident: 1.0 was a production shift, 2.0 was a learning shift, 3.0 is an infrastructure shift, and infrastructure shifts always lag the thing they carry.1

The thing they carry has arrived. Coding agents that scored under 14% on SWE-bench Verified in early 2024 are clearing 95% by mid-2026.2 Inference cost at equivalent quality has fallen more than a thousand-fold in the same window.3 The two curves run in opposite directions at comparable speed, and where they cross is where the substrate sits.

We are about to stop deploying programs and start deploying invocations. The cloud — every layer of it, from the load balancer to the line item — was not designed for that.

This essay is about what that change looks like from underneath.

02 Software 3.0, plainly ¶

Three sentences carry the whole idea.

Your prompts are programs. They have inputs and outputs, side effects, version drift, and a runtime cost per call. They are checked into repos, they go through code review, they get A/B tested in production. The fact that they look like English is a UX choice, not an ontological one.

The LLM is a computer. Not a model, not an API — a computer in Karpathy's sense, with an instruction stream (the prompt), a memory hierarchy (context, KV cache, retrieval), and a clock that ticks in tokens.4 Every architectural property of the old computer needs a new answer on this one: scheduling, caching, isolation, billing, even what it means for a program to terminate.

The autonomy slider exists. The same model can be a tab-complete, a copilot, a background agent, or a fleet. The slider is continuous, the cost curve is not, and the substrate has to absorb the discontinuity. A keystroke is cents; an agent-hour is dollars; a thousand parallel agents is the bill.

Vendors will tell you "Software 3.0" is whatever they ship, the way "AI" used to mean a spam filter. The honest test is shorter. If the artifact you wrote was a prompt, and the artifact that ran was a model, and the artifact that landed was code or a ticket or a refund, you are in 3.0.

This sits on top of a training curve that has run about five times per year since 2020 and is on track to cross 10²⁷ FLOPs by 2027.5 Four orders of magnitude in six years on the training side. A thousand-fold drop on the inference side. The substrate is the place those two curves land.

None of the three sentences is about the model. All three land on the infrastructure.

02b The Stochastic CPU ¶

A classical CPU is a deterministic function. Same input bytes, same instruction stream, same output bytes — yesterday, today, on the rack next to it, on the one in Dublin. Every property we built the industry on inherits from that single guarantee.

Cache hits because the hash of the input is the hash of the output. Tests pass or fail because the answer is fixed. Reproducible builds reproduce. Blame walks cleanly from a stack trace back to a line. Contracts hold because both sides compute the same thing from the same bytes. Sixty years of systems engineering — virtual memory, distributed consensus, exactly-once delivery, Merkle trees, the whole shape of git — is scaffolding on top of f(x) = y, always.

The LLM is a computer in Karpathy's sense.16 It takes instructions in, returns results out, and you program it in English. It is also the first computer in production that does not satisfy f(x) = y, always. It is a stochastic CPU, and that one word changes everything above it.

The break

Ask the same question twice. You get two answers. Sometimes the difference is a synonym, sometimes the variable is renamed, sometimes the control flow inverts and the function still passes its tests. Temperature is the knob you see; nucleus sampling, top-k, and the kernel's own floating-point non-associativity across batched GPU reductions are the ones you don't.17 Even at temperature=0 the output drifts, because greedy decoding over a batched matmul is not bit-exact when the batch composition changes between calls.

Then the model itself moves underneath you. A snapshot rolls. A safety fine-tune lands on Tuesday. The provider rotates a server-side seed you were never told existed. The function you were calling last week is, in a literal sense, not the function you are calling now.

Regenerate a 40-line Python file twice and diff the bytes. The first version writes for i, row in enumerate(rows):, the second writes for idx, row in enumerate(rows):. Same behavior, different SHA. Your content-addressed cache, your code review tooling, your "did anything change" check — all of them think work happened. None did. The model is doing something we don't fully name yet: producing outputs that are semantically equivalent over a space the bytes can't see.

What breaks downstream

The CPU was deterministic for sixty years. The new one isn't.

Every assumption built on determinism now needs a replacement, and each replacement is a line item on the new substrate.

Caches. Content-hash caches miss on the second call because the bytes are different even when the meaning isn't. The cache key has to move up the stack — from the output hash to the prompt prefix that produced it. This is the seed of the two-level cache in §5.

Tests. A boolean assertion over a stochastic function is a coin flip dressed as CI. Tests become evals: statistical regressions run online and offline, with confidence intervals, drift detection, and a budget. Green and red give way to a distribution.

Reproducibility. "Did this build reproduce" is replaced by "what model snapshot, prompt, system prompt, tool schema, and seed produced this artifact." The reproducibility unit is no longer the binary; it's the invocation.

Idempotency. A retried HTTP call used to be free if the server was well-behaved. A retried agent call writes a different PR, opens a different ticket, sends a different email. Every action needs an idempotency key the agent itself does not generate.

Cost. You pay for the same work, plural, until it lands. An agent loop that needs three samples to converge is three times the bill, and frontier inference is already running into the tens of millions of tokens per second per cluster. Eval is not a quality tool; it's the line item that decides whether the loop terminates before the invoice does.

Provenance. A commit used to carry one signature. It now carries three: the human who asked, the agent that wrote, and the platform snapshot that generated. Audit, compliance, and IP all have to learn to read a triple.

The punch

The substrate is not being rebuilt because LLMs got smarter. Smarter would fit on the old substrate; smarter is just a faster CPU. The substrate is being rebuilt because LLMs are stochastic, and the cloud — every layer of it, from the cache to the bill — was designed around a property the new workload does not have.

This is the most important architectural shift since the move from single machine to distributed compute, and it is happening underneath teams that still think they are shipping software.

03 What broke ¶

Containers were built for humans. Agents broke them in eighteen months.

A container is a frozen filesystem with a process boundary. It was designed for a workload that boots, serves requests, scales out, and dies — a server, kind of like a long-lived organism. Agents are the opposite shape. They wake up, think for ninety seconds, write a file, sleep, wake up again at a different hour with a different prompt and a different model. The container abstraction fights every one of those properties.

Cold start. A Python image pulls in seconds; a CUDA-enabled image with a 70B model loaded pulls in minutes. Cold start used to be a tail-latency problem. With agents it's the median.

State. A container is stateless by convention. An agent has a scratchpad, a working tree, a tool history, and a context window worth keeping. Snapshotting that across a fleet of ephemeral runners is a new primitive — one that Modal, E2B, and a half-dozen startups are now charging for, at a documented 3× premium over their own serverless tier.6

Pricing. A container bills CPU-seconds. An agent bills tokens, and the tokens are 95% of the line. A 2 vCPU sandbox costs about ten cents an hour on Modal or E2B. The same hour of Sonnet-tier inference, in a heavy loop, runs $6 to $8.50.7 The bill is the model, not the box.

CI. Tests are deterministic; agents are not. You can't gate a merge on a boolean over a stochastic function, so the gate becomes an eval — a statistical regression with a budget and a confidence interval, run on its own pool of compute. CI used to be 5% of the cloud bill. By 2030 the eval pool is its own line item.

Cache. Content-hash caches assume same input means same output bytes. Agents regenerate semantically-equivalent files with different bytes constantly, so Bazel, Buck, and Nix all miss when they shouldn't. The fix isn't a smarter hash; it's a second cache layer that keys on prompt prefix. §5.

None of this is hypothetical. Shopify reports that one in eight merged PRs across the company is coauthored by its agent, River — 3,536 merged in a 30-day window.8 Sundar Pichai said on the Q3 2024 earnings call that over a quarter of new code at Google was AI-generated; he raised the number to over 30% on the Q1 2025 call.9 The Shopify number is the more honest one because it counts what got merged, not what got typed.

The workload that broke the container is already the dominant workload on the largest engineering orgs in the world. The infrastructure has not caught up.

04 The agent-hour ¶

The agent-hour is the new CPU-hour.

Watch the billing pages, not the keynotes. The keynotes will sell you the model; the billing pages tell you which abstraction the providers actually believe in.

Anthropic's June 2026 billing split broke out agent calls as a separate SKU from raw inference for the first time. Modal added a sandbox tier at $0.1419 per physical core-hour, roughly 3× its standard serverless rate, and called it the sandbox tax in the docs.6 Replit Agent ships Core at $20/mo with $25 of credits and meters Agent runtime against the pool. Cursor's background agent has its own SKU. The pattern is the same in every case: a unit of billing that did not exist in 2023 now has a price.

The unit is the agent-hour. It is to the cloud what the CPU-hour was to EC2. It is the thing that gets reserved, the thing that gets autoscaled, the thing that ends up on the invoice. By 2027 all three hyperscalers will have an agent-hour SKU, and the one that doesn't will be the one re-selling someone else's.

The reason it works as a unit is that the substrate underneath it is finally cheap enough. The economics of a coding agent that issues hundreds of LLM round-trips per task only close because the inference curve has fallen by more than three orders of magnitude at equivalent quality. Frontier price per million output tokens has roughly halved in three years; equivalent quality has fallen more than a thousand-fold over the same window.3 The first curve is what the press release covers. The second one is what makes the agent-hour solvent.

The other half of why the agent-hour is solvent is that the work is worth doing. Devin 1.0 cleared 13.86% on SWE-bench in March 2024.10 Devin 2.0 hit roughly 45.8% on Verified by December.11 Claude Sonnet 4.5 reached 77.2% in September 2025, Claude Opus 4.7 hit 87.6% in April 2026, and Fable 5 sits at 95% as of June.2 Sometime in Q1 2026 the line crossed the human baseline. That crossing is what made SW 3.0 a deployment problem instead of a research problem.

Walk the math for a single agent-hour at mid-2026 prices. A Sonnet-tier background loop issuing 200–600 round-trips, with 30K–150K-token contexts (80–95% cached) and 2K–8K output per call, lands between $1.20 and $8.50 per hour all-in. Sandbox compute is a rounding error on top.7 A fully-loaded software engineer is $80–$150 an hour. The ratio is 10× to 100× before you count parallelism.

The substrate's job is no longer to make the agent cheap. The substrate's job is to make the agent's coordination cheap. The bottleneck has moved up the stack, which is where every interesting infrastructure problem now lives (!).

05 The two-level cache ¶

Bazel hashes inputs and reuses outputs. Buck hashes inputs and reuses outputs. Nix hashes inputs and reuses outputs. The entire build-system tradition of the last fifteen years is a content-addressed cache that assumes f(x) = y, always.

An LLM regenerating a file is the workload that breaks that assumption. Variable renames, comment churn, reordered imports, equivalent control flow — the bytes change while the meaning doesn't. The hash misses. The cache rebuilds. The bill arrives.

The fix is two caches stacked on each other. The bottom layer is the old content-hash cache — still useful, because most of the build graph is still deterministic compilers chewing on deterministic inputs. The top layer is a prompt-prefix cache. It keys on the prompt the agent issued, normalized; on a hit, you reuse the agent's last output without recomputing it, even if you would have recomputed the bytes.

Anthropic does this at the inference layer already. Prompt-cache reads are billed at 0.1× the input rate — a 90% discount — and writes are billed at 1.25× (5-minute TTL) or 2× (1-hour TTL). For Claude Opus 4.x that's $15 per million input tokens dropping to $1.50 on a cache hit.7 The catch is byte-identical prefix. The agent harness has to be disciplined about prefix stability or the whole discount evaporates.

That discipline propagates down the build system. Prompts get versioned. System prompts get pinned. Tool schemas get hashed. The "what does this build depend on" question grows a new column: not just the source files and compiler version, but the model snapshot, the prompt prefix, and the seed if you bothered to set one.

The shape of the cache is the shape of the workload. The workload changed. The cache has to.

06 The Tokio benchmark ¶

We ran cargo test --workspace on Tokio three ways. Cold: 46 seconds. Warm content-hash cache: 12 seconds. Hot two-level cache, prompt-prefix layer included: 3 seconds.13

The interesting number is not 46. Anyone who has run a Rust test suite knows 46. The interesting number is 3, because 3 is the latency an agent sees when it edits one crate out of fifty and re-runs the workspace. The agent does that constantly — that's how agents work — and a 3-second loop is a usable interactive substrate, while a 46-second loop is a coffee break.

What the two-level cache does on Tokio is collapse the dominant case. Forty-seven of fifty crates are unchanged. The bottom cache catches those. The top cache catches the agent's regenerated file as semantically equivalent to last call. The remaining work is the small set of crates that actually moved.

This is one repo. The same shape recurs on any monorepo where an agent is the dominant editor. The break-even point — where the prompt-prefix cache starts paying back the cost of running it — comes early, around the third or fourth iteration of the agent on the same codebase.

3 seconds is not a benchmark result. It's a loop latency that lets the agent stay in the call frame instead of falling out of it.

07 Provenance ¶

A commit used to be signed by a human. By 2030, in any regulated industry, it will be signed by three parties.

The human who asked. The agent identity — model, version, prompt hash, tool schema, seed. The platform attestation — which snapshot ran, on which cluster, against which evals. SLSA v1.1 already specifies the build-provenance shape. Sigstore already signs the artifacts. The EU Cyber Resilience Act already requires it for software placed on the European market. The pieces exist; they just haven't been composed into a default yet.14

The reason this becomes mandatory is not regulatory. It's IP. When 30%+ of Google's new code is AI-coauthored and Shopify is merging 3,500+ agent PRs a month, the question "who wrote this" stops being a credit-line question and starts being an indemnity question.89 A signature you can't trace is a liability you can't bound. The triple signature is what makes the merge reviewable, the audit defensible, and the IP transferable.

This is also what closes the loop on the stochastic CPU. The output is non-deterministic; the invocation is not. You can't pin f(x) = y any more, but you can pin (human, agent, platform, prompt, snapshot) → artifact, and that tuple is what the next decade's compliance regime will be written against.

Reproducibility moved up one level of abstraction. Provenance moved with it.

08 Timeline 2025 → 2030 ¶

Six dated milestones, written down so they can be checked later.

2025. Coding agents cross 75% on SWE-bench Verified. Claude Sonnet 4.5 lands at 77.2% in September.2 The autonomy slider becomes a product surface, not a research demo.

2026. Agents cross the ~85% human baseline. Fable 5 hits 95% in June. The agent-hour appears as a discrete SKU on at least one major provider's invoice. Shopify reports one in eight merged PRs are agent-coauthored.8

2027. All three hyperscalers have an agent-hour SKU. Frontier training compute crosses 10²⁷ FLOPs, on Epoch AI's central forecast.5 Large engineering orgs cross the 50% coauthored-code threshold.

2028. Power is the binding constraint, not silicon. AI's share of data-center load is climbing toward the IEA's projected 35–50% by 2030, and at least one frontier cluster is sited next to a dedicated generator.15

2029. Triple-signature provenance ships as a default in at least one major CI provider. The EU CRA enforcement deadline does the rest.14

2030. Global data-center electricity consumption hits ~945 TWh, roughly double 2024.15 Kubernetes still runs, but it has been demoted from the platform to the scheduler for the agent fleet underneath the platform.

09 The 2030 bill ¶

The shape of a cloud invoice is the most honest forward indicator we have. Money goes where the workload is, and the workload is moving.

The 2030 split, for a large engineering org running agent-heavy workflows, lands roughly here. Inference: 40%. Agent-hours, billed separately from raw inference because the coordination surface is its own product: 25%. Snapshot and cache infrastructure, including the two-level cache: 15%. Eval compute, which has eaten the role CI used to play: 10%. Egress: 5%. Legacy compute — the EC2 instances, the RDS, the old shape: 5%.

Compare that to 2023, where roughly 70% of a comparable bill was the legacy shape and inference barely cleared single digits. The reallocation is not gradual. It is the kind of shift that happens once a workload crosses a threshold, and the agent workload has crossed it.

Kubernetes survives. It just gets demoted. The control plane that schedules pods becomes the control plane that schedules agent runtimes — and the things it schedules are not stateless. They have warm KV caches, prompt-prefix locality, model affinity to specific GPU pools. The scheduler that wins this round will look more like a database query planner than the round-robin we have today.

Power is the limit. The IEA projects global data-center consumption rising from 415 TWh in 2024 to ~945 TWh by 2030, with AI taking the dominant share of the growth.15 Frontier inference clusters in 2028 will be sited around the megawatt, not the rack. The bill will reflect that, too.

This is the most consequential infrastructure shift since virtualization, and most of the people building it have never written a CUDA kernel.

10 Closing ¶

Software 1.0 was humans writing for machines. Software 2.0 was data writing weights. Software 3.0 is humans writing for a model that writes for the machine, and the model in the middle is a stochastic CPU.

Every layer above that CPU was built on a guarantee the new CPU does not provide. The cache misses. The test flakes. The build is no longer reproducible by the binary; it's reproducible by the invocation. The commit needs three signatures, not one. The invoice grows a line item with a unit nobody priced in 2023. None of this is because the model got smarter. All of it is because the model is stochastic, and the substrate underneath it was not.

What you are watching is not a tooling refresh. It is the rewriting of a sixty-year stack against a property change in its lowest layer. The container, the cache, the test runner, the build system, the scheduler, the bill — each one is being reformed around the same observation, and most teams are still writing CI files as if the answer were a boolean.

The substrate is shifting. It's already shifted. You're standing on it.
